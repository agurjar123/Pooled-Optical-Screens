## Reproducible Research Code — Best Practices

### 1. Determinism

- Always pass an explicit `seed` (or `rng = np.random.default_rng(seed)`) to any
  function that uses randomness. Never use `np.random.seed()` (global state).
- Monte Carlo simulations must expose `seed` as a parameter with a documented default.
- Functions that produce figures must produce identical output given the same inputs.

### 2. Dependencies

- Pin all dependencies in `requirements.txt` with exact versions (`==`).
- Use a dedicated virtual environment; do not rely on system-level packages.
- When adding a new dependency, add it to `requirements.txt` immediately.

### 3. Data

- Raw data in `data/` is read-only. Never overwrite or modify raw files.
- All derived / processed data goes in `outputs/` with descriptive filenames.
- CSV column names must be stable; document any parsing logic (e.g., handle stripping)
  in the function that loads the file.

### 4. Code Organization

- All reusable logic lives in `src/` as importable Python modules.
- Notebooks in `notebooks/` are for analysis and visualization only — they call
  functions from `src/`, they do not define them.
- One function per concern; avoid side effects in pure computation functions.
- Visualization functions that save files must accept an `output_path` parameter
  (pass `None` to display inline instead of saving).

### 5. Function Signatures

- Expose `seed`, `n_trials`, and `prefix_len` / `L` as explicit parameters.
- Use type hints for all public functions.
- Provide a one-line docstring at minimum; full NumPy-style docstrings for
  functions with multiple parameters or non-obvious return values.

### 6. Notebooks

- Restart kernel and run all cells before committing a notebook.
- Do not commit notebooks with stale or failed outputs.
- First cell: imports only. Second cell: data loading. Keep cells short and focused.

### 7. Output Files

- Save figures to `outputs/` with descriptive names:
  `{analysis_name}_{param}.png` (e.g., `ambiguity_vs_L_k2.png`).
- Use `dpi=150, bbox_inches='tight'` for all `savefig` calls.

### 8. Testing / Verification

- For deterministic simulations, verify that `dropout_p=0.0` in dropout simulation
  exactly matches the noiseless baseline.
- For any new analysis function, include a brief sanity check cell in the notebook.
